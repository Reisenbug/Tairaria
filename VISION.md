# Terraria Vision Agent — 项目构想

## 愿景

构建一个纯视觉驱动的 Terraria AI agent。不读取游戏内存、不依赖 API，仅通过"看屏幕"理解游戏状态并操控角色完成游戏目标。这是 General Computer Control (GCC) 在具体游戏场景中的落地尝试。

目前全球范围内，Minecraft 方向已有 Voyager、SIMA 2、Cradle 等项目，但 **Terraria 方向尚无任何公开的纯视觉驱动 agent 项目**，属于空白领域。

---

## 四层架构

系统不是一个统一循环，而是四个不同频率的层同时运行：

### 1. 大脑 — LLM（0.2-0.5 Hz，每 2-5 秒一次）

- 模型：GPT-4o / Qwen-VL / 其他 VLM（云端调用，延迟可接受）
- 职责：全局战略决策
- 输入：小脑提供的结构化 JSON 状态 + 游戏知识库 + 自身记忆
- 输出：带优先级的任务队列
- 示例输出：
  ```json
  {
    "goal": "前往丛林",
    "task_queue": [
      {"trigger": "遇到树", "action": "摇树", "priority": "high"},
      {"trigger": "遇到箱子", "action": "开箱", "priority": "high"},
      {"trigger": "木头>=10", "action": "制作平台", "priority": "medium"},
      {"trigger": "遇到草药", "action": "捡起", "priority": "low"},
      {"trigger": "default", "action": "向左移动", "priority": "baseline"}
    ]
  }
  ```

### 2. 脊髓 — 行为树 / 状态机（与小脑同频，2-5 Hz）

- 实现：纯代码逻辑（Python），不需要 AI
- 职责：根据大脑的任务队列 + 小脑的实时感知，做出即时决策
- 这一层编码的是**玩家的游戏经验**——什么时候摇树、什么怪该打什么怪该跑、什么时候放平台
- 可以覆盖（override）大脑的指令（如紧急闪避）
- 可以向大脑发送中断信号（如遭遇危险、需要重新规划）
- 示例逻辑：
  - 小脑报告前方 200px 有树 → 检查任务队列中"遇到树"是高优先级 → 暂停移动 → 走到树下 → 切斧子 → 砍树 → 捡掉落物 → 恢复移动
  - 小脑报告前方有绿色史莱姆 → 弱敌 → 切剑 → 边走边砍，不停下
  - 小脑报告双面夹击 + 血量低 → 中断所有任务 → 战斗模式 → 通知大脑

### 3. 小脑 — 轻量视觉模型（2-5 Hz，每 200-500ms 一次）

- 模型：YOLO（fine-tuned on Terraria） + 专用检测器
- 职责：从屏幕截图中提取结构化状态，**不做任何决策**
- 输出格式：固定结构的 JSON（不是自然语言描述）
- 示例输出：
  ```json
  {
    "player": {"hp": 340, "max_hp": 500, "pos": [1200, 450]},
    "enemies": [
      {"type": "slime_green", "pos": [1400, 430], "moving": "left"},
      {"type": "jungle_bat", "pos": [900, 300], "moving": "down"}
    ],
    "threats": [
      {"type": "projectile", "pos": [1300, 445], "direction": "left", "urgent": true}
    ],
    "objects": [
      {"type": "tree", "pos": [1050, 450], "distance": 150},
      {"type": "chest", "pos": [800, 460], "distance": 400}
    ],
    "terrain_ahead": "pit",
    "equipped": "iron_pickaxe",
    "hotbar": ["pickaxe", "sword", "torch", "potion", null, null],
    "inventory": {"wood": 12, "platform": 5, "torch": 3}
  }
  ```

### 4. 手 — 规则脚本（30-60 Hz，每帧执行）

- 实现：pyautogui / pydirectinput
- 职责：将脊髓的动作指令翻译成持续的键鼠操作
- 示例：收到 `{"action": "move", "direction": "left"}` → 持续按住 A 键，直到收到新指令

---

## 信息流

```
屏幕像素 → 小脑(YOLO) → 结构化JSON → 脊髓(行为树) → 动作指令JSON → 手(键鼠脚本) → 游戏
                                          ↑
                                     大脑(LLM) 每2-5秒更新任务队列
```

全程不使用自然语言传递实时信息。自然语言仅存在于游戏知识库中（预先编写）。

---

## 关键设计决策

### 为什么不用 VLM 逐帧理解画面？
- 当前最强 VLM（GPT-4o）推理一帧需要 1-3 秒，无法实时
- 自然语言描述是有损压缩，信息在两次 NL 转换中丢失严重
- 结构化 JSON 输出几乎无延迟，且下游系统可直接消费

### 为什么需要脊髓层？
- 大脑太慢（2-5s），无法处理"前方有坑要放平台"这种即时决策
- 小脑太笨，只做检测，不理解"摇树能掉水果，水果能加速"
- 脊髓填补了"感知"和"战略"之间的"战术"空白
- 脊髓的规则来自玩家的游戏经验，是项目的核心竞争力

### 先验知识的作用
- 给 agent 游戏规则和目标，难度降低 40-60%
- 不需要 agent 自己发现合成系统、自己学会打 Boss
- 游戏知识库（wiki 整理为结构化文档）作为大脑的参考

## 开发路径

### Phase 1：行为树（现在就能开始）
- 用 Python 写脊髓的行为树逻辑
- 用手动编造的假 game_state JSON 测试
- 覆盖各种场景：遇树、遇怪、遇坑、低血量、双面夹击等
- 目标：行为树在假数据上的决策像一个合格的 Terraria 玩家
- **不需要任何视觉模型或屏幕截图**

### Phase 2：视觉系统（小脑）
- 截取大量 Terraria 游戏画面
- 标注怪物、方块、弹幕、UI 元素
- 可利用 Terraria wiki 的 sprite 图自动合成训练数据（sprite 贴到不同背景上，自动生成标注）
- Fine-tune YOLO 模型
- 输出格式与 Phase 1 的假数据完全一致，行为树代码零修改接入
- Terraria 是像素风 2D 游戏，同一种怪物外观完全一致，所需训练数据远少于 3D 游戏

### Phase 3：大脑接入
- 接入 LLM 做高层规划
- 大脑读取小脑的状态 JSON + 游戏知识库，输出任务队列
- 实现大脑-脊髓的通信：任务队列更新、中断信号

### Phase 4：集成与调试
- 四层联调
- 第一个完整目标：从出生点自动前往丛林
- 逐步增加复杂度：打 Boss、合成装备、探索地牢

## 赶路场景：出生点 → 丛林（默认向左）

### 需要处理的事件
1. **摇树**：斧子按键 + 智能光标 + 捡掉落物（水果提供 5min 加速）
2. **砍树**：获取木头，用于制作平台
3. **开箱子**：路上的地表箱子，获取早期装备
4. **赶路**：越过方块/平台/坑，利用铁轨加速（技术手法），踩向日葵获得加速 buff
5. **制作物品**：木头 → 平台（越过地形障碍）
6. **收集草药/蘑菇**：用于后续探索的药水材料
7. **怪物处理**：
   - 弱怪（绿/蓝史莱姆）：边走边砍，不停下
   - 中等怪（秃鹫、丛林蝙蝠）：停下来打
   - 危险情况（多怪夹击 + 低血量）：触发紧急模式，通知大脑
8. **检测金字塔**：地标性建筑，可能有宝箱
9. **水叶草 + 沙子**：进入沙漠区域的标志
10. **地下沙漠判断方向**：确认是否偏离路线
11. **照明**：黑暗区域放置火把

## 相关项目与论文参考

- **Cradle** (BAAI/清华, 2024)：General Computer Control 框架，截图输入+键鼠输出，在 RDR2 中完成主线任务
- **SIMA 2** (Google DeepMind, 2025)：跨游戏泛化，零训练适应新游戏
- **CombatVLA** (2026)：专注动作游戏战斗，比 Cradle 快 50 倍
- **Voyager** (2023)：Minecraft agent，但依赖游戏 API 而非纯视觉
- **ScreenAgent** (IJCAI-24)：VLM 驱动的通用计算机控制 agent

## 技术栈（初步）

- **语言**：Python（行为树、集成层）
- **视觉模型**：YOLOv8 / YOLOv11（fine-tuned on Terraria sprites）
- **大脑 LLM**：GPT-4o / Qwen-VL / DeepSeek-V3（通过 API 调用）
- **键鼠控制**：pyautogui / pydirectinput
- **屏幕截取**：mss / PIL
- **训练数据生成**：Terraria sprite sheets + 自动合成
- **测试框架**：pytest（行为树单元测试）

## 项目独特优势

1. **Terraria 视觉 agent 全球首创**——Minecraft 方向已被研究，Terraria 是空白
2. **开发者本人是 Terraria 深度玩家**——行为树里的游戏智慧无法替代
3. **开发者有 Terraria 技术栈经验**——TShock 插件开发、tModLoader mod 开发、服务器运维
4. **像素风 2D 游戏降低视觉难度**——比 3D 游戏的目标检测简单一个量级
5. **解耦架构允许渐进式开发**——每一层可以独立开发和测试
